# text_style_jp - 日本語テキスト スタイリングコマンド

## 概要

`text_style_jp` は、Microsoft 日本語スタイルガイドに基づき、日本語技術文書とソースコードコメントの表記を統一するコマンドである。全角・半角の変換、単語間スペースの挿入・削除を自動で行い、可読性を向上させる。

Markdown に加えて、C/C++、C#、Python、Shell、Makefile のソースコードコメントも扱える。

## 背景

### Microsoft 日本語スタイルガイドとは

Microsoft が自社製品のローカライズ用に策定した日本語表記ガイドライン。Microsoft Language Portal からダウンロード可能で、IT 業界で広く参照されている。

参照元:

- [Microsoft Localization Style Guides](https://learn.microsoft.com/en-us/globalization/reference/microsoft-style-guides)
- [Japanese Style Guide (PDF)](https://download.microsoft.com/download/a/8/2/a822a118-18d4-4429-b857-1b65ab388315/jpn-jpn-StyleGuide.pdf)

### 類似のスタイルガイド

| ガイド | カタカナ複合語の区切り | 特徴 |
|--------|------------------------|------|
| Microsoft | 半角スペース | IT 業界標準、製品ローカライズ向け |
| IBM | 中黒 (・) | 伝統的な表記 |
| JTF (日本翻訳連盟) | 中黒 or 半角スペース | 翻訳業界標準、CC BY ライセンス |
| Apple | 区切りなし | シンプルさ重視 |

本コマンドは Microsoft スタイルを採用している。

## 実装ルール

### 文字種の変換

#### 全角英数字・括弧類 → 半角

全角の ASCII 英数字 (A-Z, a-z, 0-9) および括弧類 (（）［］｛｝) は半角に変換する。

| 変換前 | 変換後 |
|--------|--------|
| `ＮＯＴＥ` | `NOTE` |
| `１２３` | `123` |
| `Ｗｉｎｄｏｗｓ` | `Windows` |
| `（タイトル）` | `(タイトル)` |
| `［新規］` | `[新規]` |
| `｛key｝` | `{key}` |

#### 半角カタカナ → 全角

半角カタカナは全角に変換する。濁点・半濁点の結合も処理する。

| 変換前 | 変換後 |
|--------|--------|
| `ｶﾀｶﾅ` | `カタカナ` |
| `ｶﾞｷﾞｸﾞ` | `ガギグ` |
| `ﾊﾟｿｺﾝ` | `パソコン` |

### スペース挿入ルール

#### 全角文字と半角英数字の間

全角文字 (漢字、ひらがな、全角カタカナ) と半角英数字の間に半角スペースを挿入する。

| 変換前 | 変換後 | 説明 |
|--------|--------|------|
| `第3章` | `第 3 章` | 漢字と数字 |
| `Windows10を起動` | `Windows10 を起動` | 英数字とひらがな |
| `ABCあいう` | `ABC あいう` | 英字とひらがな |
| `データAを参照` | `データ A を参照` | カタカナと英字 |

このルールは Microsoft スタイルガイドの中核であり、プロポーショナルフォント環境でも視覚的な区切りを明確にする効果がある。

#### カタカナ複合語

英語で複数単語から成るカタカナ語は半角スペースで区切る。

| 変換前 | 変換後 |
|--------|--------|
| `ダイアログボックス` | `ダイアログ ボックス` |
| `メニューバー` | `メニュー バー` |
| `コントロールパネル` | `コントロール パネル` |

分割と保護の判断は、Microsoft Terminology Collection (JAPANESE.tbx) から自動生成した辞書を主軸に行う。

- tbx に **単独カタカナ語として登録された語** (例: `ワークスペース` `サブディレクトリ` `クロスプラットフォーム` `トラブルシューティング` `インライン`) は `no_space` リストとして自動登録され、分割対象から除外される。
- tbx の **スペース区切りカタカナ複合語** (例: `ダイアログ ボックス` `ファイル マネージャー`) は `add_space` ペアとして自動登録され、連結形が標準形へ分割される。
- tbx 単独語に登録されていない残りのカタカナ列のみ、SudachiPy (形態素解析、モード B) によりフォールバックで分割される。

SudachiPy が形態素として認識できない語や、`replace` 適用後に初めて分割できる語については `add_space` エントリで補う。詳細は「辞書ファイル」セクションを参照。

### スペース削除ルール

#### 句読点の前

句読点、コロン、セミコロン、疑問符、感嘆符の直前のスペースを削除する。

| 変換前 | 変換後 |
|--------|--------|
| `クリックして 、` | `クリックして、` |
| `閉じます 。` | `閉じます。` |
| `更新しますか ?` | `更新しますか?` |
| `警告 !` | `警告!` |
| `フォント :` | `フォント:` |

#### 括弧の内側

開き括弧の直後、閉じ括弧の直前のスペースを削除する。

| 変換前 | 変換後 | 括弧の種類 |
|--------|--------|-----------|
| `( タイトル )` | `(タイトル)` | 丸括弧 |
| `[ 新規 ]` | `[新規]` | 角括弧 |
| `「 test 」` | `「test」` | 鉤括弧 |

#### スペース不要の単位記号

度 (°)、パーセント (%) の前のスペースを削除する。

| 変換前 | 変換後 | 備考 |
|--------|--------|------|
| `45 °` | `45°` | 角度 |
| `50 %` | `50%` | パーセント |
| `50 ％` | `50%` | 全角パーセント→半角変換後 |

また、写真・映写関連の単位 `mm` もスペース不要とする。

| 変換前 | 変換後 |
|--------|--------|
| `35 mm` | `35mm` |

他の単位記号 (kg, m, km など) は数値との間に半角スペースを入れる規則があるが、本コマンドでは自動挿入は行わない。

### スペース挿入の補完ルール

#### 疑問符・感嘆符の後

疑問符・感嘆符の直後に半角英数字が続く場合、スペースを挿入する。

| 変換前 | 変換後 |
|--------|--------|
| `保存しますか?Excelを` | `保存しますか? Excel を` |

#### ページ番号形式 + 括弧

`数字/数字` の直後に括弧が続く場合、スペースを挿入する。

| 変換前 | 変換後 |
|--------|--------|
| `10/13(ページ)` | `10/13 (ページ)` |

### Markdown 固有の処理

#### コードブロックの保護

コードブロック (` ``` ` で囲まれた範囲) 内はスタイリング対象外とする。

```text
# 変換対象
第3章 概要

# 変換対象外
```python
print("第3章")
```
```

#### インラインコードの保護

バッククォート (`` ` ``) で囲まれたインラインコードはスタイリング対象外とする。

| 入力 | 出力 |
|------|------|
| `` `第3章`を参照 `` | `` `第3章` を参照`` |

#### URL の保護

`https://` または `http://` で始まる URL はスタイリング対象外とする。
URL 末尾が日本語句読点（`。`、`、`、`」` など）の場合、句読点は URL の一部とみなさない。

| 入力 | 出力 |
|------|------|
| `詳しくは https://example.com/日本語abc を参照` | `詳しくは https://example.com/日本語abc を参照` |
| `[text](https://example.com/日本語パス)` | `[text](https://example.com/日本語パス)` |
| `参照先 https://example.com/doc。次の章` | `参照先 https://example.com/doc。次の章` |

#### 行末 trailing spaces の正規化

Markdown では行末の半角スペース 2 つ以上がソフト改行 (line break) として機能するが、
次行が空行または文末（EOF）の場合は段落の区切りが明確なためソフト改行は不要となる。
このため、次行が非空行の本文行では行末を半角スペース 2 個に正規化し、
それ以外の行末 trailing spaces は除去する。

| 入力 | 出力 | 説明 |
|------|------|------|
| `行1` + 次行が `行2` | `行1  ` | ソフト改行として半角スペース 2 個を付与 |
| `行1   ` + 次行が `行2` | `行1  ` | trailing spaces を半角スペース 2 個に正規化 |
| `単独の段落  ` + 次行が空行 | `単独の段落` | trailing spaces を除去 |
| `行1  ` + 文末（EOF） | `行1` | trailing spaces を除去 |

コードブロック内の行は対象外とする。

## 辞書ファイル

スペース挿入ルールをプロジェクト・ユーザーごとにカスタマイズできる。

### 置き場所

以下の順に読み込み、定義はマージされる。  
同じキーの設定が存在する場合は、後から読み込まれた定義内容が優先される。

1. `~/.text_style_jp/*.json` — ログインユーザーのホームディレクトリ設定
2. `<docsfw>/.text_style_jp/*.json` — docsfw サブモジュール共通の設定
3. `<カレントディレクトリ>/.text_style_jp/*.json` — カレントディレクトリの設定

パスが重複する場合は自動的に除去される（例: `<docsfw>` がカレントディレクトリと一致する場合）。フォルダ内の `*.json` ファイルをファイル名のソート順で読み込む。

docsfw 共通辞書では、読み込み順を明確にするため `NN_name.json` 形式の 2 桁数字プレフィックスを推奨する。番号が小さい辞書ほど先に読み込まれ、同じ `from` キーが重複した場合は後ろの番号の定義が優先される。

例:

- `10_microsoft.json` — TBX から自動生成したベース辞書 (no_space、replace 順方向/逆方向、add_space)
- `20_adverb-kana.json` — 一般的なかな化ルール
- `30_testfw-keywords.json` — 製品固有の除外ルール
- `90_local-katakana.json` — TBX 未収録の社内独自カタカナ語

### フォーマット

```json
{
  "no_space": [
    "Pre-Assert確認",
    "CI/CD環境"
  ],
  "add_space": [
    {"from": "タスクマネージャー", "to": "タスク マネージャー"},
    {"from": "ダイアログボックス", "to": "ダイアログ ボックス"}
  ],
  "replace": [
    {"from": "コンピュータ", "to": "コンピューター"},
    {"from": "プリンタ", "to": "プリンター"}
  ]
}
```

| キー | 説明 |
|------|------|
| `no_space` | 単独カタカナ語の許可リスト。全角/半角境界へのスペース挿入、SudachiPy による過剰分割、`replace` の誤適用を抑制する。カタカナ列のスペース挿入誤記 (例: `サブ モジュール`) も連結形が登録されていれば自動的に補正される |
| `add_space` | スペースを挿入する変換ペアのリスト。SudachiPy が対応できない語 (`replace` 適用後に初めて分割できる語など) を補う |
| `replace` | 汎用文字列置換の変換ペアのリスト (長音記号の付与・削除、かな化など) |

`_comment` など未知のキーは無視される (コメント代わりに使用可)。

`10_microsoft.json` は `tbx_to_dict.py` により JAPANESE.tbx から自動生成され、次の内容を含む。

- **`no_space`**: tbx に単独カタカナ語として登録された語のうち長さ 3 以上 (約 2,000 件)。これにより `ワークスペース` `サブディレクトリ` `クロスプラットフォーム` `トラブルシューティング` `インライン` などが過剰分割されない
- **`replace` (順方向 ー 付与)**: tbx の `ー` で終わる単独語から、その `ー` を除いた形を `from` とするペア。例: `サーバ → サーバー`、`ユーザ → ユーザー`
- **`replace` (逆方向 ー 削除)**: tbx 単独語 `w` のうち、`w + ー` が tbx に登録されていない場合に `w + ー → w` のペアを生成。例: `カテゴリー → カテゴリ`、`メモリー → メモリ`、`プロパティー → プロパティ`。語末音が tbx 内の `ー` 付与パターンと一致する語に限定して生成し、`コンピューター` のように両方が tbx 標準形として併存する語は除外する
- **`add_space`**: tbx のスペース区切り複合語から、スペースを除いた連結形を `from` とするペア。例: `ファイルマネージャー → ファイル マネージャー`

`tbx_to_dict.py` を再実行することで JAPANESE.tbx 更新時に辞書を再生成できる。

`90_local-katakana.json` には JAPANESE.tbx に **未収録の社内独自カタカナ語** のみを登録する。例: `サブモジュール` (tbx に登録なし) を `no_space` に追加すると、`サブ モジュール` のようなスペース挿入誤記も自動的に `サブモジュール` に補正される。

辞書の処理順序は次のとおり。

1. `replace` を Sudachi 分割前に先行適用 — 逆方向 `カテゴリー → カテゴリ` や順方向 `スライドショ → スライドショー` を、Sudachi B 分割や `no_space` 保護で分断される前に処理する
2. `no_space` プレースホルダー化 (長い順マッチ)
3. SudachiPy モード B でカタカナ列を分割
4. `no_space` 復元 (カタカナ境界に自動スペース補完、ただし `ー` と `・` は境界扱いしない)
5. `replace` を再適用 (Sudachi 分割後に初めて連続する語に対応)
6. `add_space` 適用 (連結形を Microsoft 標準のスペース区切りへ)

### 注意事項

- インラインコード（`` ` `` で囲まれた範囲）内は変換対象外
- コードブロック（` ``` ` で囲まれた範囲）内は変換対象外
- `no_space` の長い単語から順にプレースホルダー化するため、部分一致による誤置換は発生しない
- 全カタカナ同士の `replace` は、前後が非カタカナまたは行境界のときだけ一致する
- `no_space` に登録した語は `replace` の直前に再保護されるため、長音付与などの置換対象にならない
- 読み込みエラー時はその辞書ファイルを無視して処理を継続する
- SudachiPy が未インストールの場合は初回実行時に自動インストールを試みる（`--user` スコープ）。インストールできない場合は `add_space` 辞書のみで動作する

## 使用方法

### コマンドライン

```bash
# Markdown ファイルを処理
python text_style_jp.py input.md --mode markdown -o output.md

# 拡張子から自動判定してコメントを処理
python text_style_jp.py input.c --mode auto -o output.c

# 標準入出力
cat input.md | python text_style_jp.py --mode markdown > output.md

# モードを明示して処理
python text_style_jp.py input.cs --mode csharp -o output.cs

# チェックのみ (CI 向け)
python text_style_jp.py --mode auto --check input.py

# テスト実行
python text_style_jp.py --test
```

### Python モジュールとして

```python
from text_style_jp import style_markdown, style_source_comments, style_prose

# Markdown 全体を処理 (コードブロック保護あり)
styled = style_markdown(text)

# C ソースのコメントだけを処理
styled = style_source_comments(source_text, "c")

# 単純なテキストを処理
styled = style_prose("第3章")  # "第 3 章"
```

## 処理順序

スタイリングは以下の順序で適用される。

1. コードフェンス・インラインコード・URL を保護 (変換対象外化)
2. カタカナ列に挟まれた半角スペースを除去 (連結形が `no_space` に登録されている場合のみ)
3. `replace` を先行適用 (Sudachi 分割や `no_space` 保護で連続性が失われる前に逆方向 ー 削除や順方向 ー 付与を処理)
4. `no_space` 語をプレースホルダー化
5. 全角英数字・括弧類を半角に変換
6. 半角カタカナを全角に変換
7. 全角スペースを半角に変換、連続スペースを 1 つに正規化
8. 全角文字と半角英数字の間にスペースを挿入
9. 句読点前のスペースを削除
10. 括弧内側のスペースを削除
11. 単位記号 (°, %, mm) 前のスペースを削除
12. 疑問符・感嘆符後に英数字が続く場合スペースを挿入
13. ページ番号形式後の括弧前にスペースを挿入
14. スペースの正規化
15. SudachiPy (モード B) でカタカナ連続部分を形態素分割
16. `no_space` 語を復元 (カタカナ境界スペースを自動補完、`ー` と `・` は境界扱いしない)
17. `replace` を適用
18. `no_space` 語を再復元 (カタカナ境界スペースを自動補完)
19. `add_space` を適用
20. URL・保護パターンを復元
21. ソフト改行と見なせない行末の不要な trailing spaces を除去 (Markdown 固有)

## 対応モード

| モード | 主な対象 |
|--------|----------|
| `markdown` | Markdown 本文。コードフェンスとインラインコードは保護する |
| `c`, `cpp` | `//`、`/* */`、Doxygen コメント |
| `csharp` | `//`、`/* */`、`///` XML ドキュメント コメント |
| `python` | `#` コメント |
| `shell` | `#` コメント |
| `make` | `#` コメント |

`--mode auto` を指定した場合は、ファイル名と拡張子からモードを推定する。

## 制限事項

### 未実装の機能

以下の Microsoft スタイルガイドの規則は本コマンドでは未実装。

| 項目 | 例 | 理由 |
|------|-----|------|
| 数字と単位間のスペース | 3kg → 3 kg | 単位の判定が困難 |
| 数値の漢数字変換 | 一つ → 1 つ | 文脈依存 |
| 長音記号の付与・省略 | コンピュータ → コンピューター、カテゴリー → カテゴリ | tbx から自動導出、辞書ファイルでも上書き可能 |

### 既知の問題

| 問題 | 説明 |
|------|------|
| 数式内の処理 | LaTeX 数式内で意図しない変換が発生する可能性 |
