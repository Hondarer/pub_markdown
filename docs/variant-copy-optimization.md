# バリアントコピー最適化

## 概要

`pub_markdown_core.sh` は言語 × 詳細レベルのマトリクスで複数の成果物 (バリアント) を生成します。ソースファイルに言語タグや詳細タグが存在しない場合、全バリアントの内容は同一です。この最適化機構はその同一性を事前に検出し、Pandoc の再実行を省略して `cp -p` のコピーに置き換えることで生成時間を短縮します。

## バリアントマトリクス

生成マトリクスは `lang` × `details_suffixes` の 2 軸で構成されます。

| 軸 | 例 | 出力ディレクトリ例 |
|---|---|---|
| 言語 | `ja`, `en` | `pages/ja/`, `pages/en/` |
| 詳細レベル | (通常), `-details` | `pages/ja/`, `pages/ja-details/` |

`lang="ja en"`, `details="both"` の場合、1 ファイルから最大 4 バリアント (`ja`, `en`, `ja-details`, `en-details`) が生成されます。

## 不変性の検出

マトリクスループに入る前にソースファイルを grep し、3 つのフラグを算出します。

| フラグ | 検出対象 | 不変判定への影響 |
|---|---|---|
| `_has_lang_tags` | 言語タグ (`<!--ja:-->` 等) | 言語軸を変化させる |
| `_has_details_tags` | 詳細タグ (`<!--details:-->` 等) | 詳細軸を変化させる |
| `_has_toc` | `\toc` コマンド | 言語軸を変化させる |

これらから 2 軸の不変性を導出します。

```
_lang_invariant    = !_has_lang_tags && !_has_toc
_details_invariant = !_has_details_tags
```

### `\toc` と詳細軸

`insert-toc.sh` は `DOCUMENT_LANG` 環境変数を参照して言語ごとに異なる見出しタイトルを取得しますが、details フラグは参照しません。そのため `\toc` は**言語軸の不変性のみ**を崩し、詳細軸には影響しません。`\toc` を含むファイルであっても、詳細タグがなければ詳細軸のコピー最適化が適用されます。

## 最適化パターン

| `_lang_invariant` | `_details_invariant` | 動作 |
|---|---|---|
| true | true | 最初の 1 バリアントのみ生成し、残りを全てコピー |
| true | false | 詳細レベルごとに最初の言語で 1 回生成し、他言語へコピー |
| false | true | 言語ごとに最初の詳細レベルで 1 回生成し、他詳細レベルへコピー |
| false | false | 最適化なし。全バリアントを生成 (従来動作) |

## 割り切り事項

### レベル 1 見出しの詳細別切り替えは非対応

レベル 1 見出し (`# タイトル`) を `<!--details:-->` タグで切り替えるケースは想定していません。

レベル 1 見出しは文書タイトルとして `md_title` に抽出され、Pandoc の `--metadata title=` に渡されます。詳細軸のコピー最適化では、コピー元バリアントの `md_title` をそのままコピー先にも適用することになります。レベル 1 見出しが詳細タグで切り替わる構造にすると、コピー先バリアントのタイトルが誤った値になります。

また、`insert-toc.sh` が参照先ファイルのタイトルを抽出する際も details を考慮しないため、`\toc` による詳細軸コピー最適化も同様の前提に依存しています。

**言語別のレベル 1 見出し切り替えは正常に動作します。** `insert-toc.sh` は言語を考慮したタイトル抽出を行い、生成処理も言語ごとに実行されるためです。

## 検出パターンの仕様

コードブロック内に記述されたタグも検出されますが、その場合は最適化が抑制されるだけで誤った出力は生じません (保守的な動作)。

言語コードはパターン `[a-z]+` で照合するため、`replace-tag.sh` の `supported_langs` に新しい言語コードを追加しても検出ロジックの変更は不要です。

## 出力例

コピーが適用されたバリアントは、コンソールに `(copy)` 付きで表示されます。

```
  > pages/ja/html/guide/setup.html
  > pages/en/html/guide/setup.html (copy)
  > pages/ja-details/html/guide/setup.html (copy)
  > pages/en-details/html/guide/setup.html (copy)
```

上記は `_lang_invariant=true`, `_details_invariant=true` の場合の例です (`ja` のみ生成、他 3 バリアントはコピー)。
